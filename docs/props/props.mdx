# Props

This section outlines the common props and types that should be used when working with the component.

## Component

<Props of={ReactGlobe} />

## `CameraOptionsProp`

Use the `cameraOptionsProp` to control the globe camera (i.e. how the globe is viewed). You can pass any number of option fields. Default options can be imported with:

```js
import { defaultCameraOptions } from "react-globe";
```

<Props of={CameraOptionsProp} />

## `FocusOptionsProp`

Use the `focusOptionsProp` to control how focused coordinates behave. You can pass any number of option fields. Default options can be imported with:

```js
import { defaultFocusOptions } from "react-globe";
```

<Props of={FocusOptionsProp} />

## `GlobeOptionsProp`

Use the `globeOptionsProp` to control the looks and behaviors of the globe. You can pass any number of option fields. Default options can be imported with:

```js
import { defaultGlobeOptions } from "react-globe";
```

<Props of={GlobeOptionsProp} />

## `LightOptionsProp`

Use the `lightOptionsProp` to control the color and intensity values of ambient and point lights. The ambient light illuminates all objects in the scene equally while the point light provides a source of light emitting from a single point. You can pass any number of option fields. Default options can be imported with:

```js
import { defaultLightOptions } from "react-globe";
```

<Props of={LightOptionsProp} />

## `MarkerOptionsProp`

Use the `markerOptionsProp` to control the looks and behavior of markers. If you provide a valid marker `type` (e.g. `bar`, `dot`), `react-globe` will render the corresponding supported marker. You can pass any number of option fields. Default options can be imported with:

```js
import {
  defaultDotMarkerOptions,
  defaultBarMarkerOptions,
  defaultMarkerOptions
} from "react-globe";
```

<Props of={MarkerOptionsProp} />

## `Animation`

An `Animation` is a Javascript object with four required fields that describe how an animation should behave. `react-globe` will sequentially execute the animation steps in an array of animations.

<Props of={AnimationProp} />

## `Marker`

A `Marker` is a Javascript object with required `coordinates` and `value` fields that describe how it should be placed and drawn on the globe. It can contain any other additional fields that are useful for rendering tooltips and passing them to pointer events.

<Props of={MarkersProp} />

## `Globe` Instance

`react-globe` provides the ability to work with the underlying `Globe` class instance (via the `Globe` named export or the `onGetGlobeInstance` prop).

Shown below is a summary of class variables and methods exposed on the `Globe` class. Note that these class interfaces and APIs are **less stable** than the component APIs. Therefore, it is **strongly recommended** that people still use the component props to control globe behaviors. That being said, the `Globe` class provides a powerful escape hatch to control globe behaviors outside of React lifecycles if you need the flexibility.
